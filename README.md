## 🚀 Featured Projects

### [Kayak AI Platform — Agentic Booking System](https://github.com/puks0618/Kayak)
*A stateful AI-driven flight booking assistant with event-driven backend and full cloud deployment.*

Built an end-to-end agentic booking platform using LangGraph and Ollama (qwen3 14B) to orchestrate multi-step user interactions, with an event-driven backend powered by FastAPI and Apache Kafka. Session context persists in Redis and itineraries are stored in MongoDB Atlas. Deployed to AWS ECS/ALB via GitHub Actions CI/CD with Kubernetes pod management.

**Key Features:**
- Stateful multi-step agentic assistant using LangGraph
- Event-driven booking workflow with FastAPI + Kafka
- Session state in Redis, itineraries in MongoDB Atlas
- Automated CI/CD: GitHub Actions → AWS ECR → ECS/ALB
- Kubernetes replica management and zero-downtime deployments

**Tech Stack:** FastAPI, Apache Kafka, LangGraph, Ollama, Redis, MongoDB, AWS ECS/ALB, Docker, Kubernetes, GitHub Actions

### [Forecast Blackout — Power Outage Risk Prediction](https://github.com/PrajwalDambalkar/DW-Project)
*An end-to-end ML pipeline for predicting power outage risk using Airflow, Snowflake, dbt, and AWS SageMaker.*

Architected a production-grade data pipeline using Amazon MWAA (Airflow) to orchestrate data extraction from Snowflake to S3, with automated dbt transformations enforcing data quality checks and schema validation. Trained and evaluated ML models in AWS SageMaker and visualized outage risk scores in QuickSight for non-technical stakeholders.

**Key Features:**
- Airflow (MWAA) pipeline with 99% reliability for daily risk assessments
- dbt transformations with data quality checks and lineage tracking
- ML model comparison: Random Forest, XGBoost, SVM in SageMaker
- QuickSight dashboards for stakeholder reporting
- Dockerized training workflows with automated dbt data quality validation

**Tech Stack:** Python, SQL, Snowflake, dbt, Apache Airflow (MWAA), AWS SageMaker, AWS S3, QuickSight, Docker

### [Large-Scale Amazon Reviews Data Pipeline](https://github.com/PrajwalDambalkar/Amazon-Reviews-Pipeline)
*A distributed ETL and analytics pipeline processing 250GB+ of Amazon review data using Apache Spark and AWS.*

Designed and implemented an end-to-end analytical pipeline using PySpark to clean, transform, and analyze 250GB+ of semi-structured Amazon review data. Enforced schema standards and data quality rules, optimized storage using Parquet partitioning on S3, and delivered insights through Redshift and QuickSight. Implemented NLP-based sentiment analysis and anomaly detection achieving 81% accuracy.

**Key Features:**
- PySpark ETL processing 250GB+ semi-structured data
- Parquet partitioning on S3 for optimized query performance
- Kafka integration for high-throughput streaming workloads
- NLP sentiment analysis and anomaly detection (81% accuracy)
- Amazon Redshift + QuickSight for analytics and stakeholder reporting

**Tech Stack:** Apache Spark, PySpark, Apache Kafka, AWS S3, AWS Redshift, Amazon QuickSight, Python, NLP

### [InsightStream — LLM-Powered KPI Narrative Engine](https://github.com/PrajwalDambalkar/InsightStream)
*An agentic RAG pipeline that auto-generates structured KPI narratives from Snowflake data.*

Built an automated reporting pipeline integrating Snowflake query outputs with LangChain and Ollama to generate structured executive KPI narratives. Applied prompt engineering, hallucination checks, and structured output validation across multi-metric datasets, reducing manual reporting effort by ~6 hours/week.

**Key Features:**
- Agentic RAG pipeline with LangChain + Snowflake integration
- Auto-generated structured KPI narratives from live data
- Prompt engineering with hallucination detection and output validation
- Reduced manual reporting effort by ~6 hours/week

**Tech Stack:** LangChain, Ollama, Snowflake, dbt, Python, RAG, Prompt Engineering

### [Voyagee — Full-Stack Property Rental Platform](https://github.com/PrajwalDambalkar/Airbnb-Clone)
*An Airbnb-style rental platform with microservices architecture, OAuth2 auth, and Kubernetes deployment.*

Built a microservices-based full-stack platform using TypeScript, React, and Node.js with secure REST APIs for search, reservations, real-time availability, and role-based authentication. Deployed to AWS EKS via GitHub Actions CI/CD. Load-tested for 30K+ concurrent users with JMeter to validate performance and calibrate autoscaling.

**Key Features:**
- Microservices architecture with TypeScript, React, Node.js
- OAuth2 / JWT role-based authentication for travellers and owners
- CI/CD: GitHub Actions → AWS ECR → EKS/ALB
- Redis distributed cache for property lookups
- JMeter load tested for 30K+ concurrent users

**Tech Stack:** TypeScript, React, Node.js, Redis, AWS EKS, Docker, Kubernetes, GitHub Actions, JMeter
